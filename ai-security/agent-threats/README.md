# Agent Threats (ASI Framework)

Agentic AI security — tool abuse, memory poisoning, exfiltration, privilege escalation via agents.

## ASI-style categories (reference map)

| ID | Theme | Example |
|----|-------|---------|
| ASI01 | Prompt injection | Indirect injection via RAG |
| ASI02 | Insecure output handling | XSS via model-generated HTML |
| ASI03 | Training data poisoning | Backdoor in fine-tune set |
| ASI04 | Model denial of service | Token flooding |
| ASI05 | Supply chain | Compromised model weights / plugins |
| ASI06 | Sensitive info disclosure | PII in completions |
| ASI07 | Insecure plugin design | Over-broad MCP tools |
| ASI08 | Excessive agency | Agent deletes data without confirm |
| ASI09 | Overreliance | Users trust harmful medical advice |
| ASI10 | Model theft | API extraction attacks |
